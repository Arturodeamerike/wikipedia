¿Qué es git rebase?

git rebase es un comando que permite mover o combinar una secuencia de commits a una nueva base de commits. Es útil para mantener un historial de commits limpio y lineal.

git rebase <base_commit>

Cómo usar git rebase

Para hacer un rebase de la rama actual sobre otra rama, usa el siguiente comando:

git rebase <nombre_rama>

Ejemplo:

git rebase main

Este comando moverá los commits de la rama actual sobre la rama main.

Rebase interactivo

El rebase interactivo permite editar el historial de commits de manera más granular. Para iniciarlo, usa:

git rebase -i <base_commit>

Ejemplo:

git rebase -i HEAD~3

Este comando iniciará un rebase interactivo de los últimos 3 commits.

En el editor, puedes elegir entre varias opciones como pick, reword, edit, squash, y fixup.

Precauciones al usar rebase

Al usar git rebase, especialmente en repositorios compartidos, ten en cuenta lo siguiente:

  • Evita hacer rebase en ramas que ya han sido publicadas o compartidas.
  • Prefiere usar git merge en situaciones donde múltiples desarrolladores estén colaborando en la misma rama.

En caso de conflictos durante el rebase, Git te permitirá resolverlos antes de continuar.
