Comandos de Git

Esta página contiene una lista de comandos comunes de Git y sus descripciones.

Lista de Comandos

  • git init

    Inicializa un nuevo repositorio de Git.

  • git clone <url_del_repositorio>

    Clona un repositorio remoto a tu máquina local.

  • git add <nombre_archivo>

    Añade un archivo a la zona de preparación (staging area).

  • git commit -m "<mensaje>"

    Confirma los cambios en la zona de preparación con un mensaje descriptivo.

  • git status

    Muestra el estado de los archivos en el directorio de trabajo y la zona de preparación.

  • git push origin <nombre_rama>

    Envía los commits de la rama local especificada al repositorio remoto.

  • git pull

    Descarga los cambios del repositorio remoto y los fusiona con la rama actual.

  • git pull --rebase

    Descarga los cambios del repositorio remoto y aplica los commits locales encima de los cambios remotos.

  • git fetch

    Descarga los cambios del repositorio remoto pero no los fusiona automáticamente.

  • git merge <nombre_rama>

    Fusiona la rama especificada con la rama actual.

  • git branch

    Lista todas las ramas locales en el repositorio.

  • git checkout -b <nombre_rama_nueva>

    Crea una nueva rama y cambia a esa rama.

  • git checkout -t origin/<nombre_rama>

    Crea una nueva rama local para rastrear una rama remota con el mismo nombre.

  • git branch -a

    Lista todas las ramas, incluidas las ramas remotas.

  • git branch -d <nombre_rama>

    Elimina la rama especificada.

  • git remote prune origin

    Elimina las referencias a las ramas remotas que ya no existen.

  • git reset --hard HEAD

    Deshace todos los cambios en el directorio de trabajo que no se han confirmado.

  • git revert <hash_commit>

    Crea un nuevo commit que deshace el commit especificado.
